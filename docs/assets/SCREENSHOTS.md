# Скриншоты UI для README и презентаций

Скриншоты не коммитятся автоматически: их добавляют после локального или staging-демо.

## Куда класть файлы

Создайте каталог (если ещё нет):

```text
docs/assets/screenshots/
```

Рекомендуемые имена:

| Файл | Содержание |
|------|------------|
| `landing.png` | Главная страница (hero + блок тарифов) |
| `documents.png` | Список документов с карточками и статусами |
| `jobs.png` | Очередь обработки (фильтр по статусу job) |
| `search.png` | Поиск с ответом и источниками |
| `chat.png` | Чат с сессиями |
| `billing.png` | План и лимиты |
| `audit.png` | Журнал аудита |

## Как вставить в README

После добавления файлов в репозиторий:

```markdown
## Скриншоты

![Landing](docs/assets/screenshots/landing.png)
```

Используйте относительные пути от корня репозитория — GitHub отображает их в превью.

## Подсказка по съёмке

- Разрешение: 1280–1600 px по ширине окна браузера достаточно для README.
- Тёмная/светлая тема: согласуйте с основным UI (сейчас светлая тема по умолчанию).
