# Глоссарий продукта (единая терминология)

Те же формулировки используются в [README.md](../README.md), UI (Next.js) и [quotas.md](quotas.md). Технические имена в API/БД могут быть на английском (`workspace_id`, `ingestion_jobs`).

| Термин в продукте | Что значит |
|-------------------|------------|
| **Рабочее пространство (workspace)** | Изолированный контур данных: документы, векторный индекс, квоты, аудит. В HTTP передаётся заголовок `X-Workspace-Id`. |
| **Владелец (owner)** | Роль с полным контролем над workspace; расширенный просмотр журнала аудита. |
| **Администратор (admin)** | Роль для операций и безопасности; как owner — расширенный аудит в UI. |
| **Участник (member)** | Роль с рабочим доступом по политике API. |
| **Наблюдатель (viewer)** | Роль преимущественно на чтение: загрузка/удаление документов, новые диалоги и отправка в чате отклоняются API; во фронте те же ограничения отражены дизейблами и плашками (см. **Команда и доступ**, сайдбар). |
| **План (free / pro / team)** | Набор лимитов workspace: запросы, токены, объём загрузок, число документов, параллельные задачи индексации, страницы PDF. |
| **Задача индексации (ingestion job)** | Фоновая обработка загруженного файла (парсинг, фрагменты, векторы); в API/БД — **job** со статусами `queued` → `processing` / `retrying` → `ready` \| `failed` (канон: `backend/app/constants/ingestion.py`). В UI: раздел **«Очередь обработки»** (`/jobs`). |
| **Статус документа / фаза пайплайна** | Те же ключи статуса, что у job, где применимо: в UI единые подписи («В очереди», **«Индексация»** для `processing`, «Повторная попытка», «Готово», «Ошибка»). |
| **Квота** | Ограничение плана; при превышении — отказ с записью в аудит (`quota.denied`) и логах. |
| **Журнал аудита** | Записи событий безопасности и действий в рамках workspace. |
| **Запрос (в квоте)** | Учитываемое действие: поиск, сообщение в чате или загрузка документа в пределах месячного лимита плана. |
| **Команда и доступ (UI)** | Страница **`/w/:slug/team`**: роль, матрица возможностей, участники и ожидающие приглашения из API (создание / отзыв / повтор письма, смена ролей по политике владельца/админа). |
| **План и лимиты (UI)** | Страница **`/w/:slug/billing`**: план workspace (free / pro / team), статус подписки (в т.ч. grace / past_due), Stripe Checkout и Customer Portal при включённом биллинге, usage и ledger. |

См. также: [IMPLEMENTATION_STATUS.md](IMPLEMENTATION_STATUS.md), [WORKSPACE_ROUTING.md](WORKSPACE_ROUTING.md).
