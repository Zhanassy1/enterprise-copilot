## Смотреть продукт за 1 минуту

**Цель:** зритель понимает, *где* смотреть демо, без догадок.

| Путь | Действие |
|------|----------|
| **A. Только визуал** | В корневом **[README.md](../README.md#screenshots)** — таблица PNG (`docs/assets/screenshots/`). Детали съёмки: **[SCREENSHOTS.md](assets/SCREENSHOTS.md)**. |
| **B. Руки на UI** | `docker compose up --build` → **http://localhost:3000** → на главной гостя блок **«Демо за одну минуту»** (`/#demo-quick-1min`) → **Регистрация** → убедиться, что выбран **workspace** → **Документы** (пустой каталог или загрузка, если роль не viewer). |
| **C. Видео** | Запись по скрипту ниже; выложить ролик → `NEXT_PUBLIC_DEMO_VIDEO_EMBED_URL` на маркетинговой главной; дублировать embed в **[README.md](../README.md#demo-video)**. |

Текст для спикера (**45–60 сек**, голос поверх скриншотов): *«Это multi-tenant copilot: своё рабочее пространство (workspace), роли владелец / администратор / участник / наблюдатель, документы индексируются в фоне — задачи индексации (jobs), поиск и чат с источниками, план и квоты на странице плана и лимитов, журнал аудита. В репозитории уже есть скриншоты или docker compose — см. README.»*

<a id="evaluator-walkthrough"></a>

## Пятиминутная оценка (как в README)

Дублирует **[README.md → «Быстрая оценка за 5 минут»](../README.md#evaluator-five-minutes)** для тех, кто держит открытым только каталог `docs/`.

1. `docker compose up --build` → **http://localhost:3000** (маркетинг и **/#demo-quick-1min**) → при необходимости **/pricing** (Free / Pro / Team).
2. **Регистрация** → **Вход** → в сайдбаре выбрано **рабочее пространство (workspace)**.
3. **Документы** → загрузка файла → **Очередь обработки**: статусы задачи индексации **queued → processing / retrying → ready | failed** и статус документа в каталоге.
4. **Поиск** или **Чат** — вопрос по содержимому, проверка **источников** в ответе; при **наблюдателе (viewer)** — без загрузки и без отправки в чате (см. UI и API).
5. **План и лимиты** — план workspace и расход квот; **Аудит** — события; сверка с [quotas.md](quotas.md) при необходимости.

Визуальный контур без Docker: [SCREENSHOTS.md](assets/SCREENSHOTS.md) и таблица PNG в корневом README.

## Скрипт живого демо (~4 минуты)

Ориентир: **1080p**, без чувствительных данных. Порядок совпадает с **evaluator flow** в [README.md](../README.md#evaluator-five-minutes).

| Время | Шаг | Что говорить / показать |
|------|-----|-------------------------|
| 0:00 | **Контекст** | Коротко: multi-tenant copilot, RAG, workspace, async ingestion, квоты, аудит. |
| 0:20 | **Landing + pricing** | Главная гостя → прокрутка value prop → **/pricing** — планы Free / Pro / Team. |
| 0:45 | **Login / workspace** | Регистрация или вход → убедиться, что в шапке/сайдбаре выбрано **рабочее пространство**. |
| 1:00 | **Документы + upload** | **Документы** → **Загрузить** PDF/DOCX/TXT → закрыть тост, карточка со статусом. |
| 1:30 | **Async processing** | **Очередь обработки**: задача queued → processing → ready; вернуться в документы — **Готово**. |
| 2:00 | **Поиск** | **Поиск**: запрос по факту из файла → блок ответа и **источники** (чанки). |
| 2:30 | **Чат** | **Чат**: новый диалог, вопрос по содержимому → ответ с опорой на документы. |
| 3:00 | **Summary** | В **Документы** — иконка **Краткое содержание** на готовом файле → диалог summary. |
| 3:30 | **План и usage** | **План и лимиты**: тариф workspace, счётчики месяца; связка с [docs/quotas.md](quotas.md). |
| 3:50 | **Аудит** | **Аудит**: последние события (логин, загрузка, поиск — по факту среды). |
| 4:00 | **Закрытие** | Ссылка на GitHub, README, раздел «Быстрая оценка за 5 минут». |

Сокращённый вариант (**3 мин**): пропустить pricing или детальный разбор очереди; показать только переход «загрузил → готово → один запрос в поиске и в чате».

## Видео-демо (слот на сайте и в README)

Пока ролика нет, в UI и README остаётся аккуратный **placeholder** — как только появится запись, тот же блок превращается в плеер без переделки страницы.

1. Снимите экран по таблице сценария (~4 мин) или по пятиминутному гиду выше.
2. Выложите на YouTube / Vimeo / корпоративный CDN.
3. **Маркетинговая главная:** `NEXT_PUBLIC_DEMO_VIDEO_EMBED_URL` = полный **URL iframe** (YouTube: `https://www.youtube.com/embed/VIDEO_ID`).
4. **Только README:** вставьте тот же iframe в секцию «Демо-видео», чтобы GitHub показывал ролик тем, кто не открывает локальный UI.

```html
<iframe width="560" height="315" src="https://www.youtube.com/embed/VIDEO_ID" title="Enterprise Copilot demo" allowfullscreen></iframe>
```

Качество: **1080p**, **3–6 минут**, без чувствительных данных (или с явным дисклеймером).
